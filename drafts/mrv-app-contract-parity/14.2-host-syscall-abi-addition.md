# DRAFT — §14.2 Host syscall ABI addition (not published)

> **Status: DRAFT. Do not publish.** Proposed addition to §14.2 of the whitepaper.
> Insert after the existing syscall list and the "consensus contract" paragraph,
> before §14.3. The existing §14.2 text is unchanged; this only adds the
> host-context syscalls.

---

The base syscall list above is stable today. A second, narrow group of
**host-context syscalls** extends the ABI so that application logic ported from
other ecosystems sees the block and call metadata it expects, without the chain
executing foreign bytecode:

```text
block_timestamp() -> u64
chain_id() -> u64
call_value() -> u128
```

- `block_timestamp()` returns the committed anchor's timestamp in seconds. This
  is **not** a host wall clock — the value is fixed by consensus and identical for
  every operator that replays the block, so execution stays fully deterministic.
  The "no wall clock" guarantee in §14.1 is preserved: the timestamp is part of
  the agreed anchor context, not a reading taken from the operator's machine.
- `chain_id()` returns the network's chain identifier, letting a contract reject
  cross-network replay without hard-coding a network into its code.
- `call_value()` returns the native value attached to the current call frame, so
  a contract can inspect the value transferred alongside the call it is currently
  executing.

These host-context syscalls are part of the consensus contract on the same terms
as the rest of the ABI. They are introduced through a coordinated upgrade and
become active at a foundation-signed milestone height; before that height they
are not part of the active ABI. Clients and contracts detect activation through
the node capability surface rather than assuming a fixed height.

Alongside the host-context syscalls, the runtime emits a uniform rich receipt
for every contract action. A bare contract deploy — which would otherwise carry
no call-style receipt — is given a synthesized receipt sidecar carrying the
assigned contract address, code hash, whether the constructor ran, metered gas
and cycles, storage-write count, emitted events, and status. Explorers, wallets,
and SDKs then read deploys and calls through one receipt shape.

## Where this fits the rest of the section

No change is needed to §14.1 (the artifact's import table already declares the
host syscalls a contract uses, so these appear there when used) or §14.3 (the
cost model already meters host syscall cost as a distinct resource; the
host-context syscalls are cheap reads of already-committed context).
